# Самостійна робота

## Тема роботи

Розробка вебінтерфейсу для взаємодії з NLP-сервісом.

## Мета роботи

Метою самостійної роботи є створення вебінтерфейсу, який забезпечує зручну взаємодію користувача з REST-сервісом NLP. Інтерфейс має дозволяти вводити текстові запити, надсилати їх до backend-сервісу, отримувати результати NLP-обробки, переглядати історію діалогу, перевіряти статус інтеграції з LLM та оцінювати продуктивність сервісу.

## Хід роботи

### 1. Створення структури вебінтерфейсу

На першому етапі було створено окремий frontend-застосунок `nlp-web-interface-self-work`, який містить HTML-сторінку, CSS-стилі та JavaScript-логіку. Такий підхід дозволяє не ускладнювати проєкт додатковими frontend-framework залежностями та швидко перевіряти роботу інтерфейсу разом із Spring Boot NLP-сервісом.

### 2. Налаштування з’єднання з NLP-сервісом

У вебінтерфейсі реалізовано поле для введення base URL backend-сервісу. За замовчуванням використовується адреса `http://localhost:8080`. Також додано кнопку перевірки доступності сервісу через endpoint `/actuator/health/readiness`. Це дає змогу переконатися, що backend працює перед виконанням NLP-запитів.

### 3. Реалізація форми обробки тексту

Було створено форму для введення довільного тексту та надсилання його до endpoint-а `/api/text/process`. Після обробки на сторінці відображаються нормалізований текст, токени, кількість токенів та інша інформація, яку повертає NLP-сервіс.

### 4. Реалізація класифікації наміру

Для перевірки моделі класифікації намірів було додано розділ, який працює з endpoint-ами `/api/intent/status` та `/api/intent/classify`. Користувач може перевірити стан OpenNLP-моделі та виконати класифікацію нового текстового повідомлення.

### 5. Побудова інтерфейсу діалогової моделі

У вебінтерфейсі реалізовано створення діалогової сесії, надсилання повідомлень користувача та завантаження історії діалогу. Для цього використовуються REST endpoint-и `/api/dialogue/sessions`, `/api/dialogue/sessions/{id}/messages` та `/api/dialogue/sessions/{id}/history`. Це демонструє взаємодію UI з діалоговою моделлю та збереженням історії у базі даних.

### 6. Інтеграція з LLM endpoint-ами

Окремий розділ інтерфейсу дозволяє перевірити статус LLM-інтеграції через `/api/llm/status` та виконати генерацію відповіді через `/api/llm/generate`. Якщо ключ API не налаштований, backend повертає fallback-відповідь, що також відображається у UI.

### 7. Оцінка продуктивності NLP pipeline

Для перевірки стабільності та швидкодії було додано форму запуску performance test через endpoint `/api/performance/nlp`. Користувач задає тестовий текст і кількість ітерацій, після чого отримує метрики часу виконання.

### 8. Виведення JSON-результатів

Для зручності перевірки та налагодження кожна відповідь REST API додатково виводиться у вигляді форматованого JSON. Це дозволяє бачити повну структуру відповіді backend-сервісу.

### 9. Контейнеризація вебінтерфейсу

Для запуску вебінтерфейсу у контейнері було створено `Dockerfile`, конфігурацію `nginx.conf` та `docker-compose.yml`. Nginx використовується для роздачі статичних файлів і проксіювання запитів `/api` та `/actuator` до Spring Boot backend-сервісу.

## Висновки

У результаті виконання самостійної роботи було створено вебінтерфейс для взаємодії з NLP-сервісом. Реалізований UI дозволяє виконувати обробку тексту, класифікацію наміру, роботу з діалоговою моделлю, перевірку LLM-інтеграції та оцінку продуктивності NLP pipeline. Вебінтерфейс забезпечує зручний спосіб тестування REST API без прямого використання Postman та демонструє практичне застосування NLP-сервісу у користувацькому середовищі.
